Docker必须要清楚的6个问题：

1.什么是Docker？
Docker是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包到一个可移植的容器中，然后发布到任何流行的Linux机器上也可以实现虚拟化。

2.为什么要使用Docker？
(1) Docker 的启动很快。ps：较传统的虚拟机方式启动速度会很快。
(2) Docker 对系统的资源利用率很高。ps:一台主机可以运行多个docker容器。
(3) Docker 性能好，开销小。
(4) Docker 节约时间和成本。

3.Docker具备哪些优势？
(1) 更快速的交付和部署。（敏捷）
(2) 更高效的虚拟化。（不需要额外的hypervisor支持）
(3) 更轻松的迁移和扩展。
(4) 更简单的管理。

4.什么是Docker镜像？
Docker 镜像就是一个只读的模板。
镜像可以用来创建 Docker 容器。
Docker提供了一个很简单的机制来创建镜像或者更新现有的镜像，用户甚至可以直接从其他人那里下载一个已经做好的镜像来直接使用。

5.什么是Docker容器？
Docker 利用容器来运行应用。
容器是从镜像创建的运行实例。它可以被启动、开始、停止、删除。每个容器都是相互隔离的、保证安全的平台。

6.什么是Docker仓库？
仓库是集中存放镜像文件的场所。
仓库分为公开仓库（Public）和私有仓库（Private）两种形式
最大的公开仓库是 Docker Hub，存放了数量庞大的镜像供用户下载。国内的公开仓库包括 Docker Pool等，可以提供大陆用户更稳定快速的访问。
当用户创建了自己的镜像之后就可以使用 push 命令将它上传到公有或者私有仓库，这样下次在另外一台机器上使用这个镜像时候，只需要从仓库上 pull 下来就可以了。
